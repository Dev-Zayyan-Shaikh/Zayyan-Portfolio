01 / Concept

Multi-Approach AI Diagnostics & Wallet Transaction Gating.

Technical Summary:

  • Allopathic, Ayurvedic, & Homeopathic Outputs
  • OpenAI GPT-4o-mini diagnostic prompts
  • Vercel Serverless Functions backend API
  • Firebase Firestore NoSQL Database

The Clinical Software Paradigm

Patients looking for digital medical analysis often have to choose between Western (Allopathic), Traditional (Ayurvedic), or Homeopathic systems. RoboDoc AI unifies these practices. Running entirely as a web software platform, the application lets users input their symptom profiles, runs parallel AI diagnostic prompts on the gpt-4o-mini engine, and returns a structured multi-view analysis containing home remedies and warnings.

To manage API overheads, RoboDoc AI features a credit-based coin wallet system. Users buy coin packages via Razorpay checkout, and balances sync instantly on Firebase Firestore. Smart tools like the AI Chef, AI Exercise planner, and AI Nutritionist allow customized routines.

01 / AI Reasoning Pipeline

Multi-School AI Diagnostic Engine

Main symptom analysis core. Processes patient-provided metrics (symptom description, age, gender, severity, weight, height) across three distinct medical perspectives: Allopathic, Ayurvedic, and Homeopathic. The engine runs these parameters through separate specialized system prompts using the OpenAI GPT-4o-mini API to generate suggestions, remedies, and safety warnings.

02 / Custom Tokenomics

Credit-Based Health Wallet

Configured a credit economy system managing user access logs. A full symptom check deducts 10 coins, follow-up questions cost 1 coin, and specialized health tools cost 2 coins. Balance deductions are validated by backend serverless rules and synced dynamically in Firestore NoSQL.

03 / Transaction Gateway

Integrated Razorpay Payments

Enables card transactions to buy credit coin packs. Implements Razorpay Checkout overlays on the vanilla JS client, calling secure Node.js Vercel Serverless Functions to verify the cryptographic order signature using the Razorpay Node SDK before updates.

06 / Engineering Details

Engineering Challenges & Resolutions

1

Managing Multiple Prompt Roles

**Challenge**: Routing symptom parameters concurrently to Western, Ayurvedic, and Homeopathic prompt contexts can cause overlapping parameters and model output formats confusion.

**Resolution**: Designed isolated prompt template wrappers in serverless functions. Each approach is processed by independent GPT-4o-mini API calls with distinct systemic safety rails before layout assembly.

2

Asynchronous Wallet Verification

**Challenge**: Synchronizing Firestore token balances during rapid diagnostic calls could allow race conditions where users bypass coin deduction checks.

**Resolution**: Implemented transaction locks. The Vercel function utilizes Firestore database transactions, ensuring that coin balances are decremented and confirmed on the server before starting AI model requests.

3

Serverless Latency Overheads

**Challenge**: Cold start delays in Node.js serverless functions on Vercel latency-spikes the initial user diagnosis query up to 3 seconds.

**Resolution**: Structured lightweight function bundles. Minimized server-side package dependencies and cached Firebase connection parameters in global scope memory to keep execution under 400ms.
